Preprocessing Insights

Summarized from the notebook: data cleaning steps, feature engineering, and anomaly handling for customer segmentation.

Dataset Preparation

The notebook loads and inspects customer_info, checks column types, and prepares a clean dataset for clustering.

Missing Values

Missing entries are identified and handled carefully. Future transaction years are set to NaN and later imputed instead of dropping rows.

Invalid Data Fixes

Negative promotion percentages and future years are corrected. Constant identifiers such as loyalty card numbers are removed from modeling.

Feature Engineering

Education level is extracted from customer name prefixes like Bsc., Msc., Phd., then names are standardized to protect privacy.

Outlier Detection

Spending features show strong right skew and clear outliers. IQR capping is used to limit the influence of extreme values.

Behavioral Patterns

Temporal patterns such as typical transaction hour are preserved using angle-based transformations to keep cyclic proximity.
