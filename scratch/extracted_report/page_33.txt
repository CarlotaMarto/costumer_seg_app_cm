Self-Organising Maps add a fourth, topologically-flavoured perspective: instead of partitioning the data directly, a SOM first arranges customers onto a 2-D grid such that nearby grid cells contain similar customers, and then each input feature can be visualised as its own “heat map” over that grid (Figures 28a–b). Reading these feature planes side by side is a powerful sanity check — wherever, say, the alcohol-spend plane lights up in the same region as the promotion-usage plane, that is direct visual evidence that those two behaviours travel together in the customer base, which is exactly the kind of co-occurrence a good segment should be built around. Figures 29a–b then show the SOM's own cluster structure (its U-matrix / unit-cluster view), which the notebook compares back to the KMeans partition. The SOM broadly confirms the same regions of behavioural similarity that KMeans found, adding yet another independent vote in favour of the chosen segmentation — at this point the notebook has now corroborated its solution with hierarchical clustering, an R²-based quantitative benchmark, and a topology-preserving neural method, in addition to the original silhouette/elbow/dendrogram trio used to pick k. 6.12 From cluster labels to business meaning
