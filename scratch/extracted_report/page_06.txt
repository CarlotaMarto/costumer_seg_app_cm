The missing-value barplot (Figure 1) shows a single dominant outlier: loyalty_card_number is missing in roughly 40% of records. Every other column with missing data — lifetime_spend_fish, lifetime_spend_electronics, lifetime_spend_videogames, lifetime_spend_meat, lifetime_spend_vegetables, typical_hour, lifetime_spend_petfood, number_complaints, distinct_stores_visited, percentage_of_products_bought_promotion, teens_home, lifetime_spend_alcohol_drinks, kids_home, lifetime_spend_hygiene and customer_birthdate — sits below roughly 3%. The interpretation drawn is sound: the dataset does not need wholesale row-dropping, because missingness is concentrated and shallow enough to be handled with imputation (later resolved with KNN imputation in Notebook 1), while loyalty_card_number's ~40% gap is treated separately because the variable itself turns out to be closer to an identifier than a behavioural feature (its uniqueness is checked directly, see Section 4). 3.3 Data-entry errors and statistical summary A targeted integrity check on percentage_of_products_bought_promotion finds values outside the valid [0, 1] range — a textbook data-entry error for a ratio variable, flagged here and fixed later in Notebook 1 (Section 2.3, “Fixing Negative Percentages”). The .describe() table surfaces two more useful observations: loyalty_card_number behaves like a near-constant identifier with no behavioural value for clustering, and several variables — most visibly the spend categories — show heavily right-skewed distributions with a long tail of high-value customers.
